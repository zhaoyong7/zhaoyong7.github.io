获取php cli模式里的参数方法

发表于 2015-06-25   |  

php可以在cli（命令行）模式里运行，不需要人和一种web服务器做载体

在cli模式里如何获取传入的参数呢？

1、使用全局参数$argv接受参数（register_argc_argv打开时可用）

cli ： /php/xxx.php one two three

print_r($argv)

打印：array(
            [0] => /php/xxx.php
            [1] => one 
            [2] => two 
            [3] => three 
        )

第一个参数为文件路径，其余为传入参数（$argc为参数个数，所以最小为1）

2、使用getopt()函数获取参数

cli ： /php/xxx.php －a one -b two -c three

//获取 -a -b -c参数
$args = getopt('a:b:c:');
print_r($args);

打印：array(
            [a] => one 
            [b] => two 
            [c] => three 
        )

####1、apache配置:
找到httpd.conf中的这两段内容：

 # AllowOverride controls what directives may be placed in .htaccess files.
# It can be "All", "None", or any combination of the keywords:
#   Options FileInfo AuthConfig Limit
#
AllowOverride none

将AllowOverride none 改为 AllowOverride all

#LoadModule rewrite_module libexec/apache2/mod_rewrite.so

将#去掉，打开rewrite模块

####2、.htaccess文件:
写入下面内容,并放入yii2根目录的web目录下：webroot/yii2/web(yii2是自己命名的项目名称，webroot的Apache根目录)

Options +FollowSymLinks
IndexIgnore /
RewriteEngine on
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . index.php

####3、config/web.php配置urlManager:

'urlManager' => [
        'enablePrettyUrl' => true,
        'enableStrictParsing' => true,
        'showScriptName' => false,
        'rules' => [
            [
                'class' => 'yii\rest\UrlRule',  'controller' => 'user' ,
            ],
        ],
    ] ,

访问路径：http://loacalhost/yii2/web/users(yii自动讲名词变为复数形式，具体看文档，可取消)

####4、controller继承ActiveController:

namespace app\controllers;
use yii\rest\ActiveController;
class CountryController extends ActiveController
{
    public $modelClass = 'app\models\Country';
}

###额外：不同modules下urlmanger配置:
假设生成了一个modules/v1 的模块，urlmanger配置如下

'urlManager' => [
       'enablePrettyUrl' => true,

       'showScriptName' => false,
       'rules' => [
           [
               'class' => 'yii\rest\UrlRule',  'controller' => ['v1/country'] ,

       ],
   ] ,

访问路径：http://loacalhost/yii2/web/v1/countries(yii自动讲名词变为复数形式，具体看文档，可取消)

UIkit Dynamics(UIkit动力世界)

发表于 2015-03-23   |  

UIkit Dynamics是UIkit的一套基于现实世界物理驱动的一套交互体系，包含了我们常见的物理知识现象，如重力、碰撞、弹力等效果。而UIKit Dynamice提供了一整套逼真的物理动作，在实现一些逼真的物理效果的时事倍功半。

妈蛋，太专业，反正就是可以让一些不明真相的群众觉得你很牛B！

好，进入正题，让我们来用swift练习一下效果

有四个概念我们来稍微描述一下：

  • UIDynamicItem 就是物理体系可以影响到的物体，像现实世界的桌子、椅子。在ios我们常见的UI控件像UIButton、UIView等实现了该接口。
  • UIDynamicBehavior 一个行为，像重力或者弹性、吸附等效果
  • UIDynamicAnimator 一个动画执行者，所有的行为都可以加入这个执行者，重力+碰撞就可以模拟高空坠落拉！
  • referenceView 参照体系，就像现实世界的参照物。

一、重力行为UIGravityBehavior

var animator : UIDynamicAnimator!
override func viewDidLoad() {
    super.viewDidLoad()
    //box
    let box : UIView = UIView(frame: CGRectMake(135, 80, 50, 50))
    box.backgroundColor = UIColor.grayColor()
    self.view.backgroundColor = UIColor.whiteColor()
    self.view.addSubview(box)
    //参照物为背景view
    self.animator = UIDynamicAnimator(referenceView: self.view)
    //为box添加重力行为
    let gravity = UIGravityBehavior(items: [box])
    //执行动画
    self.animator.addBehavior(gravity)
}

这里注意把animator声明为类的成员变量，不然animator在viewDidLoad后就释放内存引用，就没有什么重力效果了。

######GIF效果图

我们可以设置重力的向量，大小和方向就是坠落的方向和下坠加速度了。

gravity.gravityDirection = CGVector(dx: 0, dy: 5)

二、碰撞行为UICollisionBehavior

就这么无休止的坠落深渊未免也太无聊了吧，我们来让那个它跟屏幕下方进行碰撞，让它以一定角度落下让碰撞更加明显。

######GIF效果图

三、连线行为UIAttachmentBehavior

连线就是两个物体之间进行连接或者一个物体跟一个锚点之间连接。

参数说明：UIAttachmentBehavior(item: offsetFromCenter: attachedToAnchor: )

  • item:物理体系可以影响到的物体
  • offsetFromCenter:物体中心偏移
  • attachedToAnchor:跟物体相连的锚点
var animator : UIDynamicAnimator!
override func viewDidLoad() {
    super.viewDidLoad()

    animator = UIDynamicAnimator(referenceView: self.view)

    // Do any additional setup after loading the view, typically from a nib.
    //静止的box
    let staticBox = UIView(frame: CGRectMake(50, 50, 50, 50))
    staticBox.backgroundColor = UIColor.redColor()
    staticBox.layer.cornerRadius = 5 ;
    //下坠的box
    let dynamicBox = UIView(frame: CGRectMake(250, 50, 50, 50))
    dynamicBox.backgroundColor = UIColor.grayColor()
    dynamicBox.layer.cornerRadius = 5 ;

    //为dynamicBox添加重力行为
    let cravity = UIGravityBehavior(items: [dynamicBox])
    //为dynamicBox添加连线行为
    let attachment = UIAttachmentBehavior(item: dynamicBox, offsetFromCenter: UIOffsetMake(0, 0), attachedToAnchor: CGPointMake(staticBox.center.x, staticBox.center.y))
    //为dynamicBox添加碰撞行为
    let collision = UICollisionBehavior(items: [dynamicBox])
    collision.translatesReferenceBoundsIntoBoundary = true

    self.view.addSubview(staticBox)
    self.view.addSubview(dynamicBox)

    animator.addBehavior(cravity)
    animator.addBehavior(attachment)
    animator.addBehavior(collision) 
}

四、吸附行为UISnapBehavior

很有趣的“牛皮糖”效果

var animator : UIDynamicAnimator!
let tapGesture = UITapGestureRecognizer()
var box:UIView!  
override func viewDidLoad() {
    super.viewDidLoad()

    //添加手势，轻触self.view，
    self.tapGesture.addTarget(self, action: "tappedView")
    self.view.userInteractionEnabled = true ;
    self.view.addGestureRecognizer(tapGesture)

    self.animator = UIDynamicAnimator(referenceView: self.view)

    // Do any additional setup after loading the view, typically from a nib.
    //物体
    self.box = UIView(frame: CGRectMake(110, 50, 100, 100))
    box.backgroundColor = UIColor.grayColor()
    box.layer.cornerRadius = 5

    self.view.addSubview(box)
}

//手势回调方法
func tappedView(){
    let point = self.tapGesture.locationInView(self.view)
    self.animator.removeAllBehaviors()
    let snap = UISnapBehavior(item: self.box, snapToPoint: point)
    self.animator.addBehavior(snap)
}

四、推力UIPushBehavior

为物体施加一个外力，遵循力的合成

var box : UIView!
var animator : UIDynamicAnimator!
var push : UIPushBehavior!    
let tapGetsture = UITapGestureRecognizer()

override func viewDidLoad() {
    super.viewDidLoad()
    self.animator = UIDynamicAnimator(referenceView: self.view)

    //self.view加入轻触手势
    self.tapGetsture.addTarget(self, action: "tappedView")
    self.view.userInteractionEnabled = true
    self.view.addGestureRecognizer(tapGetsture)

    //box
    self.box = UIView(frame: CGRectMake(110, 234, 100, 100))
    self.box.layer.cornerRadius = 5 ;
    self.box.backgroundColor = UIColor.grayColor()

    //加入box
    self.view.addSubview(self.box)
    //碰撞
    let collision = UICollisionBehavior(items: [box])
     collision.setTranslatesReferenceBoundsIntoBoundaryWithInsets(UIEdgeInsets(top: self.topLayoutGuide.length, left: 0, bottom: self.bottomLayoutGuide.length, right: 0))
    //推力
    self.push = UIPushBehavior(items: [self.box], mode: UIPushBehaviorMode.Instantaneous)
    self.push.angle = 0
    self.push.magnitude = 0 ;

    //加入行为
    self.animator.addBehavior(self.push)
    self.animator.addBehavior(collision)
}

//轻触手势
func tappedView(){

    let tapPoint = self.tapGetsture.locationInView(self.view)
    let centerPoint = CGPointMake(self.box.center.x, self.box.center.y)

    //角度和大小
    let angle =  atan2(tapPoint.y - centerPoint.y, tapPoint.x - centerPoint.x)
    var distance = sqrtf(powf(Float(tapPoint.x - centerPoint.x),2.0) + powf(Float(tapPoint.y - centerPoint.y),2))
    distance =  min(distance, 100.00)

    self.push.setAngle(angle, magnitude: CGFloat(distance/100))

    //每次更改push属性时，push都会变为不可活动状态，需要手动更改active为true
    self.push.active = true ;
    }

    • make

策略模式

发表于 2015-01-15   |  

策略模式是讲一组特定的行为和算法封装成类。以适应某些特定的上下文环境。这种模式叫做策略模式。

实际应用中，当一个系统有多种不同类型的用户，展示行为相似，但展示内容不同，那么我们就可以使用策略模式。如针对不同性别用户或不同年龄层用户展示不同的商品内容。

####策略模式

/*策略接口*/
interface IUserStrategy{
    function showAd() ;
    function showCategory() ;
}

/*策略类*/
class FemaleUserStrategy implements IUserStrtegy{
    function showAd(){
        echo "新款女装" ;
    }

    function showCategory(){
        echo "女装" ;
    }
}

class ManUserStrategy implements IUserStrtegy{
    function showAd(){
        echo "新款手机" ;
    }

    function showCategory(){
        echo "电子产品" ;
    }
}

/*客户端，IUserStrtegy可不继承，随意*/
class Client implements IUserStrtegy{
    private $strtegy = null

    function setStrtegy(IUserStrtegy $strte){
        $this->strtegy = $strte ;
    }

    function __construct(IUserStrtegy strte){
        $this->strtegy = $strte ;
    }

    function showAd(){
        $this->strtegy->showAd() ;
    }

    function showCategory(){
        $this->strtegy->showCategory() ;
    }
}

/*根据条件加入策略类，以后多种情况只需添加多一个策略类即可*/
$obj = $_GET['female'] ? new FemaleUserStrategy() : new ManUserStrategy() ;
$client = new Client($obj) ; // $client->setStrtegy($obj)
$client->showAd() ;
$client->showCategory() ;

适配器模式

发表于 2015-01-15   |  

适配器模式可以将截然不同的函数接口封装成统一的API

在实际应用中，如 PHP的数据库操作接口有mysql,mysqli,pdo三种，我们可以通过适配器模式统一成一致的api，方便外部调用。
类似的场景还有memcache,redis,file,apc等不同的缓存函数。

####适配器模式

/*适配器接口*/
interface IDatabase{
    function connect($host,$user,$passwd,$dbname) ;
    function query($sql) ;
    function close() ;
}

/*mysql*/
class Mysql implements IDatabase{
    function connect($host,$user,$passwd,$dbname){
        //Mysql操作接口connect代码
    }

    function query($sql){
        //Mysql操作接口query代码
    }

    function close(){
        //Mysql操作接口close代码
    }

}

/*mysqli*/
class Mysqli implements IDatabase{
    function connect($host,$user,$passwd,$dbname){
        //Mysqli操作接口connect代码
    }

    function query($sql){
        //Mysqli操作接口query代码
    }

    function close(){
        //Mysqli操作接口close代码
    }
}

/*pdo*/
class Pdo implements IDatabase{
    function connect($host,$user,$passwd,$dbname){
        //Pdo操作接口connect代码
    }

    function query($sql){
        //Pdo操作接口query代码

        //Pdo操作接口close代码
    }
}

###1.属性监视器的巧妙运用：
在做ios开发的时候，我们经常有这样的需求，当一个属性，更改时，相对应的UI控件表现出来的状态也要改变，最明显的就是，当我们做一个计时器，显示分钟和秒数需要格式化，传统我们是在把实际秒数和视图表现写在回调函数中。这样当我们有多个回调函数时，代码就很冗余了。

我们可以用更swift的方式来编写我们的代码。

var changeSecond : Int = 0 {
     willSet(newSecond){
         let min = newSecond / 60
         let second = newSecond % 60 
         labbel!.text = NSString(format:"%02d:%02d",min,second)
     }    
 }

###2.空合运算符:??

空合运算符(a ?? b)将对可选类型a进行空判断，如果a包含一个值就进行解封，否则就返回一个默认值b.这个运算符有两个条件:

  • 表达式a必须是Optional类型
  • 默认值b的类型必须要和a存储值的类型保持一致

相当于a != nil ? a! : b
