代理模式

发表于 2015-01-07   |   分类于 程序员世界   |  

代理模式是在客户端与实体之间建立一个代理对象(proxy),客户端对实体进行操作全部委派给代理对象，隐藏实体的具体实现细节。

代理对象(proxy)还可以与业务代码分离，部署到另外的服务器。业务代码中通过RPC来委派任务。

####代理模式：

/*主题接口*/
interface IUser {
    public function getName() ;
}

/*真实类*/
class UserImpl implements IUser{    
    private $name = '' ;
    public __construct($name){
        $this->name = $name
    }

    public function getName(){
        return $this->name ;
    }
}

/*代理类*/    
class UserProxy Implements IUser{
    private $impl = null ;
    public __construct(IUser $userImpl){
        $this->impl = $userImpl ;
    }

    public function getName(){
        return $this->impl->getName() ;
    }
}

/*客户端*/
class Client{
    static function main(){
        $proxy  = new UserProxy(new UserImpl('Qmore')) ;
        echo $proxy->getName() ;
    }
}

Client::main()
// Qmore

代理模式与装饰模式、监视者模式有相似处，但代理是控制真实对象的访问，而后两者为对象添加额外的功能和触发额外的事件。

代理模式在很多情况下都非常有用,特别是你想强行控制一个对象的时候,比如:延迟加载,监视状态变更的方法等等
