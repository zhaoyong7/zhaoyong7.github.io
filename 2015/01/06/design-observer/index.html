观察者模式

发表于 2015-01-06   |   分类于 程序员世界   |  

当一个对象状态发生改变时，依赖它的对象全部收到通知，并执行相对应的裸机操作，这样的情况我们可以使用观察者模式。

场景：一个事件发生后，要执行一连串逻辑操作。传统的方式是在该事件发生后过程式的侵入处理逻辑代码。一旦这些代码越来越多，会便得难以维护。这种方式是耦合的，侵入不友好的，每次增加新的逻辑需要修改事件主体代码。

观察者模式实现场景的解耦，非侵入式的通知与更新机制。

传统方式：

class Event{
    function trigger(){
        echo "do something..." ;

        echo "逻辑1" ;

        echo "逻辑2" ;

        echo "逻辑3" ;

    }
}

$ev = new Event; 
$ev->trigger() ;

观察者模式:

//观察者接口
interface Observer(){
    function update($args = null) ;
}

//观察者1号 2号 3号 等
class Ob1 implments Observer{
    function update($args = null){
        echo "逻辑1"
    }
}

//事件生成器
abstract EventGenerator {
    private    $observers = array() ;
    function addObserver(Observer $ob){
        $this->observers[] = $ob ;
    }
    function notify(){
        foreach($this->observers as $ob){
            $ob->update()
        }
    }    
}

//事件主体
class Event extends EventGenerator {
    function trigger(){
        echo "do something" ;
        //执行其他逻辑
        $this->notify() ;
    }
}

 /***控制器代码***/

 $event = new Event() ;
 //添加观察者
 $event->addObserver(new Ob1()) ;
 $event->addObserver(new Ob2()) ;
 $event->addObserver(new Ob3()) ;
 //执行主体
 $event->trigger() ;
