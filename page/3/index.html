Ajax跨域post问题

发表于 2014-12-10   |   分类于 程序员世界   |  

大多数跨域AJAX请求我们都使用JSONP ，参考 JSONP的简单原理

而jsonp是基于get请求的，当要传输的数据过大，只能基于POST请求，get形式无法满足我们的需求

可以使用CORS(跨域资源共享，Cross-Origin Resource Sharing)

CORS可以帮助我们解决AJAX跨域的问题，允许一个域的网络应用向另一个域提交跨域请求，要实现该功能十分简单，只需要由服务器返回一个相应header头，以php服务器为例子:

接口中包含以下头信息:

header(“Access-Control-Allow-Origin: *”)
//(*)表示允许任何域名向该服务器提交信息，也可以指定对应域名:
header(“Access-Control-Allow-Origin: http://www.fortest.com”)

jsonp的简单原理

发表于 2014-11-05   |   分类于 程序员世界   |  

jsonp 全称是 JSON with padding ,是为了解决跨域请求资源问题的产生的解决方案。

当我们需要在我们编写的代码内去请求其他域名的接口，但由于著名的同源策略，会因为安全问题而遭到拒绝。

$.ajax({
    url:’http://baidx.com/teach.php’,
    type:’get’
}) ;

页面会报一个这样的错误：XMLHttpRequest cannot load http://test.com/books.php. Origin http://localhost is not allowed by Access-Control-Allow-Origin.

JSONP的原理:

虽然有同源策略的限制，但是并不是所有HTML资源都必须在同一个域名，我们经常会在HTML标签中嵌入的一些图片、js、iframe、css样式等都可以通过外链的形式加载，可以利用标签跨域请求来实现。JSONP正式利用Script标签实现跨域问题的。

代码实现:

服务器端:

<?php 
    $data = array(
    array( ‘id’=>123,’name’=>’煮酒论西游’),
    array(‘id’=>124,’name’=>’封神榜’) );
    $json = json_encode($data) ;

    $callbackName = $_GET[‘callback’] ;
    echo “$callbackName($json)” ;

客户端:

<script>
    function showbooks(json){
        console.log(json)
    }
</script>
<script src = “http://baidx.com/teach.php?callback=showbooks”></script>

下面的script会返回 showbooks({xxxxxxx}) ,相当于回调了showbooks方法，实现了跨域请求问题。
